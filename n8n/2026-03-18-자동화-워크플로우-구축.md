### Context
n8n을 활용하여 자동화 워크플로우를 생성했습니다. 이 워크플로우는 디스코드(Discord)에서 메시지를 수신하면 트리거가 발생하고, 특정 에이전트(agent)가 실행되어 내용을 TIL 형식으로 변환합니다. 그런 후, 변환된 코드를 GitHub에 푸시(push)한 뒤 README.md 파일에 인덱스를 갱신하여 완료 메시지를 발신합니다.

### Core
- **Trigger Node**: Discord 노드가 메시지를 수신하여 트리거를 발생시킵니다.
- **Agent Execution**: 수신된 메시지를 TIL 형식으로 변환하는 에이전트가 실행됩니다.
- **GitHub Push**: 변환된 코드를 GitHub에 푸시합니다.
- **README Update**: README.md 파일에 인덱스를 추가합니다.
- **Notification**: 완료 메시지를 발신합니다.

```javascript
{
 "nodes": [
   {
     "name": "Discord Trigger",
     "type": "discordTrigger",
     "events": ["message"],
     "actions": [{ "path": "/agent", "action": "TIL Conversion" }]
   },
   {
     "name": "GitHub Push",
     "type": "githubPush",
     "repository": "your-repo/TIL",
     "files": ["new-til.md"]
   }
 ]
}
```

### Insight
n8n을 처음 사용하여 워크플로우를 생성해 보았습니다. 복잡해 보이지만 노드에 어떤 입력이 필요하고 어떤 출력을 생성하는지를 이해하면 쉽게 워크플로우를 구성할 수 있습니다. 이러한 자동화를 통해 업무 효율성을 크게 향상시킬 수 있습니다.
